# 火柴AI 客户端逆向与单机化 产品愿景

## 一句话愿景

把丢失源码的火柴AI v3.6.7.0 客户端还原为可读源码谱，并在客户端本地改成双击免登录、授权状态恒可用、业务功能照常联网的自用学习版。

## 目标用户与他们的真实处境

目标用户是项目作者本人。当前只有打包后的 Windows 客户端资产，源码、原服务端、云服务器、数据库和管理后台控制权均已遗失，也没有可用的旧虚拟机、历史账号或历史凭据；客户端仍需要尽可能保留号码采集、群发、云手机、投屏、视频等业务能力。

## 表层诉求 vs 根因

- 表层诉求：去掉登录、授权、时效、支付回传等阻断，让客户端能直接进入主界面。
- 根因：手里只有混淆和加密后的编译产物，且原服务端控制权和历史登录态均不可恢复，必须在不破坏 native 签名和仍可直连业务的前提下，恢复足够可读的源码谱，并为遗失的鉴权、产品、菜单及后续失效接口提供兼容实现。

## 解法与差异点

采用静态反编译加局部动态调试的方式，先建立可读源码树和字符串映射，再以 v33 为冻结技术基线，分两层恢复：M4A 从原包资源、反编译代码和国际化字典静态恢复九产品与菜单；M4B 将已经嵌入客户端的本地登录、`getInfo`、产品、菜单和 Web 首屏兼容响应正式化。后续按依赖分类保留客户端直连能力，并只为确实依赖遗失服务端的接口逐项重建兼容后端。

## 成功定义

- 断网状态下双击客户端，不出现登录页、激活页或套餐页，能进入主界面并看到 9 套系统入口。
- 8 个已开通系统可进入；独立站系统明确显示“未开通/进入官网了解”。
- 各系统侧边栏使用原包可证实的产品 code、图标和文案拓扑；未知数值 ID 使用稳定、可替换的本地兼容 ID，并明确标注为“恢复值”。
- 主界面保留授权外表，但恒显示“已授权/永久”或等价状态。
- 联网状态下，号码采集等业务功能至少跑通一次完整流程。
- 抓包验证授权、订单、支付状态类请求不再发出，或在客户端本地被静默短路。
- 产出 `string_map.json`、`seams.md`、`assets.md`、`verify.log` 和人话版业务模块文档。

## 非目标 / 明确不做

- 不尝试取得、猜测或伪造原远端服务器认可的 token，不探测或压测当前在线资产。
- 不动 `libmytrpc` 以及任何 native 签名逻辑，只把它们作为黑盒复用。
- 不追求完全离线，业务功能需要联网时继续联网。
- 兼容后端不是全量重写；只按客户端既有接口契约逐项补齐遗失能力。
- 不公开传播、不商用，逆向产物只用于自用与学习。

## 硬核 FAQ

- Q：逆不出完整源码是不是项目就做不了？
- A：不是。原生层和部分加密逻辑可以作为黑盒保留，只要能定位授权握手接缝并保持业务调用链不被破坏，就能达成本期目标。
- Q：最大技术风险是什么？
- A：字符串解密失败导致关键路径不可读，或鉴权接缝定位错误导致黑屏、闪退、误伤业务联网。降险方式是隔离虚机、快照、逐步验证、抓包区分授权请求与业务请求。
- Q：只改客户端能恢复全部业务吗？
- A：不能承诺。客户端直连第三方平台的功能可继续保留；依赖遗失服务端数据库、任务队列、云设备或算力的功能，需要逐模块重建兼容后端。
