## ADR-0001

- 编号：ADR-0001
- 标题：去授权采用“鉴权点短路返值”而非删除登录模块
- 状态：accepted
- 日期：2026-06-20
- 背景（Context）：客户端存在 ProGuard 混淆和字符串/资源加密，登录、套餐、鉴权与启动流程耦合；服务器不可动，业务联网和 native 签名链路必须保留。
- 决策（Decision）：定位唯一“鉴权结果判定点”，让它恒返 ok，并提供永久或足够远的 `expireAt` 与全解锁 `plan`；不删除登录模块，不跨模块大改。
- 取舍（利弊，放弃了什么）：获得最小改动面、可回滚性、静默性和对业务链路的低影响；放弃快速删除 UI、封域名、全量重写等表面更直接但风险更高的做法。代价是必须先完成字符串解密和接缝定位，前期分析成本更高。
- 后果（Consequences）：后续所有去授权改动必须围绕 `design.md` 第 3 节的授权握手接缝；若需要改变接缝契约，必须新增 ADR 并经用户批准。
- 取代关系（Supersedes）：无

## ADR-0002：Java 桌面 JAR 反编译采用 CFR 导出源码树
- 状态：accepted
- 日期：2026-06-20
- 背景（Context）：M1-02 计划名为 JADX 导出源码树，但只读检查发现 data/app/App.dll 文件头为 PK，内部包含大量 .class 文件，是 Java 桌面 JAR 形态而非 Android dex/apk。JADX 主要面向 Dalvik/Android 输入，当前本机也没有可用 JADX。
- 决策（Decision）：M1-02 的实际源码树导出采用 CFR 0.152 在备份副本上反编译 App.dll/JAR；JADX 不作为本轮主力工具。
- 取舍（利弊，放弃了什么）：获得与 Java class 输入更匹配、可用现有 Java 8 运行、输出源码树稳定的路径；放弃严格按原计划使用 JADX 的字面要求。代价是 CFR 对混淆和字符串加密不会自动语义恢复，后续 M2 仍需单独处理字符串解密和重命名。
- 后果（Consequences）：M1-02 输出目录和命令记录以 CFR 为准；上下文中仍保留 JADX 原计划名称，但验证报告必须说明工具替换原因。若后续发现 dex/apk 资产，再另行评估 JADX。
- 取代关系（Supersedes）：无
