# 任务拆解

## 里程碑 1：Walking Skeleton

### 任务卡 M1-01：资产备份与原始启动记录

- 目标：在不修改原始资产的前提下，建立备份点并记录客户端原始启动行为。
- 输入：仓库根目录中的安装包、DLL、`data/`、图标和配置资源。
- 输出：备份清单、资产清单草稿、原始启动行为记录。
- DoD（可自检的完成定义）：能说明每个关键资产的位置和备份位置；原始客户端启动结果有截图、日志或文字记录；未修改原始文件。
- 明确不做：不反编译、不 patch、不联网压测、不碰服务器。
- 规模（S·M·L）：S

### 任务卡 M1-02：JADX 导出源码树

- 目标：把客户端 Java/JAR 产物导出为可浏览源码树。
- 输入：M1-01 备份后的客户端产物。
- 输出：反编译源码目录、反编译命令记录、初步入口类候选。
- DoD（可自检的完成定义）：源码树可打开；能定位或列出 `StartApp`、`JLoginNew`、`ClawWorkspace` 等候选路径；不修改 native 组件。
- 明确不做：不做字符串解密、不改授权逻辑、不做 UI 删除。
- 规模（S·M·L）：M

### 任务卡 M1-03：原始 JAR/客户端可运行验证

- 目标：确认未改动产物在隔离环境中能运行，并记录启动到鉴权阻断的路径。
- 输入：原始备份产物、隔离环境、启动日志或控制台输出。
- 输出：`verify.log` 初版，包含 before 行为和鉴权阻断位置线索。
- DoD（可自检的完成定义）：能复现原始启动；能描述启动成功或失败发生在哪一环；无服务器探测或压测行为。
- 明确不做：不尝试绕过授权，不制造大量失败请求。
- 规模（S·M·L）：S

## 后续里程碑任务卡

### M2：字符串解密

- 目标：批量产出 `string_map.json`，让授权相关关键词可检索。
- 输入：反编译源码树、解密函数候选、运行时上下文。
- 输出：密文到明文映射、关键类明文化记录。
- DoD（可自检的完成定义）：关键入口明文可读；失败路径明确切换到动态 dump。
- 明确不做：不 patch 授权，不动业务联网。
- 规模（S·M·L）：M

### M3：授权接缝清单

- 目标：定位授权、时效、支付状态判定点。
- 输入：`string_map.json`、调用链、抓包关键词。
- 输出：`seams.md`，包含类.方法、返回类型、原逻辑、拟改动、回滚点。
- DoD（可自检的完成定义）：接缝精确到行或可 patch 的方法级别。
- 明确不做：不实际改动产物。
- 规模（S·M·L）：M

### M4：隔离环境最小授权补丁

- 目标：只在鉴权结果判定点短路返回 ok。
- 输入：`seams.md`、备份产物、隔离虚机快照。
- 输出：补丁产物、diff、回滚点、断网启动验证记录。
- DoD（可自检的完成定义）：断网直进主界面；授权外表恒为已授权/永久。
- 明确不做：不删登录模块，不改服务器，不改 native。
- 规模（S·M·L）：M

### M5：业务联网回归

- 目标：证明去授权没有误伤采集、云手机和 native 签名链路。
- 输入：补丁产物、抓包记录、业务测试账号或可用流程。
- 输出：`verify.log` 联网验证段、授权请求与业务请求分类。
- DoD（可自检的完成定义）：联网采集跑通一次；授权请求不发出或本地短路；业务接口无签名错误。
- 明确不做：不压测，不采集无关数据，不扩大业务功能改动。
- 规模（S·M·L）：M

### M6：业务模块文档

- 目标：把逆向理解整理成人话版业务模块文档。
- 输入：源码树、明文映射、调用链、验证记录。
- 输出：`assets.md`、模块说明、后续重构建议。
- DoD（可自检的完成定义）：每个主要模块有职责、入口、依赖和风险说明。
- 明确不做：不启动全量重写。
- 规模（S·M·L）：M

## 依赖关系

- M1 是所有后续工作的入口。
- M2 依赖 M1 的源码树和入口候选。
- M3 依赖 M2 的明文映射。
- M4 依赖 M3 的精确接缝清单。
- M5 依赖 M4 的补丁产物。
- M6 依赖 M2 到 M5 形成的源码理解和验证记录。

## 范围外

- 修改、探测或压测服务器。
- 修改 `libmytrpc`、FFmpeg、VECore 等 native 组件。
- 通过 hosts 或全局封网方式粗暴断网。
- 删除登录 UI 或大面积重构启动链路。
- 完全离线化、全量重写、公开传播或商用。
