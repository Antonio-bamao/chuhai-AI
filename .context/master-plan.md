# 火柴AI 客户端逆向与单机化 总体计划

## 目标

- G1 免授权启动：双击直进主界面，不出现登录、激活或套餐页。
- G2 去时效：不再被有效期或剩余天数拦截。
- G3 去支付回传：授权、订单、支付状态类请求不再依赖远端验证。
- G4 保留业务联网：优先保留客户端直连第三方平台的号码采集、群发、云手机、投屏、视频等能力。
- G5 兼容后端：对依赖遗失服务端数据库、任务队列、云设备或算力的接口逐项重建，不伪造远端 token。
- G6 可读源码谱：产出语义重命名、注释和业务模块文档。
- G7 静默推进：不让远端服务器产生异常告警或大量失败请求。

## 非目标 / 明确不做

- 不继续猜测、生成或伪造原远端服务器认可的 token。
- 不探测、不压测当前在线资产。
- 不动 `libmytrpc`、FFmpeg、VECore 等 native 组件和签名逻辑。
- 不追求完全离线，只做授权层单机化。
- 本期不做全量重写。
- 不传播、不商用。

## 阶段与里程碑

### M1：Walking Skeleton / 反编译环境搭建

- 范围：环境备份，JADX 全量反编译导出源码树，跑起原始客户端或原始 JAR。
- 验收：能打开源码树；能记录原始启动行为；能定位启动入口。

### M2：字符串解密

- 范围：识别字符串/资源解密方式，批量产出 `string_map.json`。
- 验收：`StartApp`、`JLoginNew`、`ext.j2026.*` 等关键路径能看到可检索明文。

### M3：授权接缝定位

- 范围：用明文关键词和调用链定位授权、时效、支付状态相关接缝。
- 验收：产出 `seams.md`，接缝精确到类、方法、返回类型、原逻辑和回滚点。

### M4A：静态恢复九产品与侧边栏

- 状态：完成。v40 已通过九产品卡、原始 logo、WhatsApp 11 项侧边栏和原始 icon 的宿主运行验收。
- 范围：冻结 v33，从原始资源、反编译代码和国际化字典恢复 8 个可进入系统、1 个未开通独立站系统及各系统菜单拓扑。
- 数据规则：原包可证实的 code、资源和文案原样使用；未知数值 ID 使用稳定、可替换的本地兼容 ID，在证据文档标为“恢复值”。
- 验收：九产品卡结构正确，8 个系统可进入，独立站保持未开通；临时 AIGC 产品和菜单撤除。

### M4B：正式化兼容鉴权、产品和菜单服务

- 状态：约 90%。免操作本地登录、产品选择器和主界面已通过；进程级黑洞代理启动已通过，尚缺物理断网和最终双击分发包验收。
- 范围：在 v33 已有嵌入式微型兼容响应基础上，正式化本地登录、`getInfo`、产品初始化、Web token bridge、`getRouters`、菜单及首屏必要响应。
- 验收：双击免操作启动；断网可进入完整主界面；授权恒可用且不依赖付费、审核和原服务端。

### M5A：验证仍可直接联网的业务

- 状态：已启动静态分类；尚未执行真实业务动作。
- 范围：逐模块分类并验证客户端直连第三方、native 依赖和原后端依赖。
- 验收：形成“客户端直连第三方 / 原后端代理 / 原后端数据库 / 云资源算力 / native 依赖”矩阵，至少跑通一个低风险真实业务流程。
- 当前产物：`.context/m5a-business-dependency-inventory.md` 第一版。

### M5B：逐项重建失效的原后端接口

- 范围：对 M5A 证明确实依赖遗失服务端的接口，按客户端现有请求/响应契约逐项实现兼容后端。
- 验收：每个重建模块有独立契约、测试、回滚点和业务验收；不把 `/prod-api/*` 泛化成无证据的通用代理。

### M6：业务模块文档

- 范围：按模块整理资产、职责、调用链和未来重构提示。
- 验收：产出人话版业务模块文档，支撑后续干净重构。

## 验收标准

- 断网状态下双击客户端，不出现登录页，直进主界面。
- 主界面保留授权外表，授权状态恒为已授权/永久。
- 联网状态下业务功能跑通至少一个完整采集流程。
- 抓包证明授权、订单、支付状态类请求不再发出或已被本地短路。
- `string_map.json`、`seams.md`、`assets.md`、`verify.log` 和业务模块文档齐备。
- 所有改动都有备份、diff、验证记录和回滚点。
- v33 JAR/ISO 哈希保持不变，后续版本从其能力集合向前演进，不回退渲染、URL、Web 首屏和 `JSBFMain` IM 配置修复。
